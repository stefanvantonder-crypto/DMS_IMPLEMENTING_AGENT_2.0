# Version Roadmap

This roadmap controls sequencing. Do not implement later-phase functionality before the active phase is approved.

## V1.x Markdown-First Static Prompt Control

### V1.0

Repository control, schemas, registries and Markdown-first baseline.

### V1.1

Agent definitions and input/output model.

### V1.2

Template manifest and minimum Markdown templates.

### V1.3

Static UI shell.

### V1.4

Controlled prompt generation.

### V1.5

Session report export.

### V1.6

V1 smoke testing and hardening.

## Future Versions

### V2.0

Optional Markdown-to-DOCX conversion.

### V3.0

Strict DOCX template population only after Markdown generation is stable.

## Development Priority Rule

Finish and validate each V1 phase before starting later phases. Backend, database, DOCX, and advanced automation features remain locked until explicitly approved by the project owner.
