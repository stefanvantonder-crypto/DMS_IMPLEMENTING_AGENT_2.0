# V1 Known Limitations

V1 is intentionally limited to the static Markdown-first prompt-control baseline.

## Known Limitations

- No backend.
- No server-side document processing.
- No DOCX generation.
- No PDF/DOCX extraction.
- No source document upload into the app.
- No permanent storage.
- No `localStorage`.
- No AI API integration.
- No automatic client deliverables.
- Browser download/import behavior depends on normal browser support.
- Actual prompt output quality depends on the uploaded source documents and ChatGPT/Codex execution.

## User Responsibility

Users must:

- Provide actual source documents manually to ChatGPT/Codex.
- Check generated outputs for accuracy, completeness and evidence strength.
- Confirm missing information is marked as `To be confirmed`.
- Confirm only selected outputs are generated.
- Save outputs manually in approved Markdown locations when appropriate.

## Release Boundary

These limitations are accepted V1 boundaries. They are not V1 defects unless a documented V1 control fails inside the approved static Markdown-first scope.
