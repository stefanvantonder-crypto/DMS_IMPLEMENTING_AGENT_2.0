# V2.4 DOCX Fidelity Evaluation Model

## 1. Purpose

Define how a future DOCX proof-of-concept would be evaluated for fidelity against the approved Word template baseline.

## 2. Fidelity Dimensions

- Section completeness
- Heading hierarchy
- Numbering
- Fonts
- Colours
- Spacing
- Headers/footers
- Tables
- Images/visuals
- Annexures
- Page layout/readability

## 3. Evaluation Method

- Compare packaged DOCX to approved Word template baseline.
- Perform visual review.
- Use checklist scoring.

## 4. Scoring Model

- Pass
- Pass with minor issues
- Fail

## 5. Non-Negotiable Failures

- Broken template styles
- Missing mandatory sections
- Dropped visuals
- Unreadable tables
- Broken numbering
- Incorrect headers/footers

## 6. Evidence Required

- Reviewer checklist
- Comparison notes
- Screenshots if appropriate

## 7. Mandatory Formatting-Control Note

Future DOCX evaluation must respect the user's mandatory DMS Word formatting controls and style requirements.
