# V2.3 Word Template Application Model

## 1. Purpose

Define how an approved Word template should be selected and checked before manual packaging begins.

## 2. Approved Template Selection Model

- Select the correct approved template for the output type.
- Confirm the template version before packaging.
- Confirm the template owner before packaging.
- Confirm the template has been approved for use.

## 3. Template Metadata To Confirm Before Packaging

- Template name
- Template owner
- Template version
- Output type
- Required styles
- Required sections
- Required visuals

## 4. Packaging Rules

- Word template remains source of styling.
- Do not overwrite template styles.
- Do not simplify client templates.
- Do not remove mandatory sections.
- Do not use `final` unless approval evidence exists.

Mandatory formatting-control note:

The user has mandatory Word style and formatting requirements for DMS templates, including Calibri/Arial/Times New Roman complex script settings, dark blue numbered heading styles, Title/Subtitle settings and Header/Footer tab settings. These must be treated as strict controls in any future packaging or DOCX automation design.

## 5. Section Transfer Method

- Title page
- Document control
- Main numbered headings
- Tables
- Review checklist
- Sign-off placeholders

## 6. Common Failure Modes

- Broken numbering
- Wrong heading styles
- Lost headers/footers
- Dropped tables
- Missing visuals
- Inconsistent fonts

## 7. Review Checkpoints

- Template selection confirmed
- Template version confirmed
- Required styles confirmed
- Required sections confirmed
- Visuals confirmed
- Tables confirmed
- Annexures confirmed
- Packaging-readiness evidence confirmed
