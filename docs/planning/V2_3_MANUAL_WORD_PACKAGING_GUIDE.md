# V2.3 Manual Word Packaging Guide

## 1. Purpose

Plan the controlled manual process for taking reviewed Markdown outputs into Word templates while preserving template integrity, formatting rules, visuals and evidence.

## 2. Current V1/V2 Baseline

Current baseline:

- V1.19C confirmed that V1 can produce usable Markdown-first outputs.
- V2.0 planning established Markdown as the canonical source format.
- V2.1 planning hardened output schema and review controls.
- V2.2 planning defined controlled Markdown capture/storage workflow and evidence chaining.
- DOCX generation is not approved.
- Backend is not approved.
- V2 coding is not approved.
- V1 static app must remain stable and protected.

## 3. Problem Statement

The project needs a safe, manual Word packaging process that turns reviewed Markdown into client-usable Word documents without converting the app into a DOCX generator.

Without a controlled packaging guide:

- Word templates may be altered unsafely.
- Styles may be broken or simplified.
- Visuals and tables may be lost.
- Packaging may be mistaken for client approval.

## 4. Definition Of Manual Word Packaging

Manual Word packaging means:

- Taking reviewed Markdown outputs that are already approved for packaging.
- Copying or transferring content section-by-section into an approved Word template.
- Preserving template styles and required formatting controls.
- Checking tables, visuals and annexures before release.

## 5. What Manual Word Packaging Does Not Mean

- Not automatic DOCX generation
- Not backend processing
- Not template auto-population
- Not final client approval

## 6. Proposed Future Workflow

- Reviewed Markdown selected.
- Packaging readiness confirmed.
- Approved Word template selected.
- Content transferred section-by-section.
- Styles preserved from template.
- Tables checked.
- Visuals inserted/repositioned.
- Annexures attached/updated.
- Packaging review completed.
- Packaging evidence recorded.

## 7. Roles

- Preparer
- Reviewer
- Approver
- Template Controller
- Packaging Controller

## 8. Control Points

- Selected-output-only confirmation
- Approved-for-packaging status confirmation
- Template version confirmation
- Style preservation check
- Visual completeness check
- Table formatting check
- Annexure completeness check
- Packaging evidence check

## 9. Planning-Only Implementation Boundary

This guide is planning only.

No DOCX implementation is approved in V2.3.
No Word packaging automation is approved in V2.3.

## 10. Acceptance Criteria For Moving To V2.4 Decision Gate

Do not move beyond planning until:

- Reviewed Markdown workflow is stable.
- Packaging-readiness evidence model is accepted.
- Template application model is accepted.
- Visual/table/annexure controls are accepted.
- V1 remains unchanged.
- No automation is implied by this guide.
